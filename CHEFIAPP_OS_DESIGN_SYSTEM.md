# CHEFIAPP™ OS — DESIGN SYSTEM OPERACIONAL

**Status**: SOBERANO (Fonte Única de Verdade)
**Versão**: 1.0 (Fire System)

---

## 0. Princípio-Mãe (A Lei)

> **ChefIApp™ não "parece" um app.**
> **Ele se comporta como um sistema em operação.**

Tudo que for decorativo, genérico ou "marketing" deve ser eliminado.
Nós não construímos interfaces; construímos ferramentas de comando.

---

## 1. Logo & Presença Visual (Soberania)

O logo representa o **OS (Sistema Operacional)** que governa a cozinha.

### Regra de Ouro
ChefIApp governa o backstage. O Restaurante brilha para o cliente.

### Onde o Logo DEVE estar (O Território do OS)
*   [x] Landing Page (Hero + Header)
*   [x] Login / Auth
*   [x] Onboarding (Ritual de Criação)
*   [x] Dashboard (Top bar discreta, como marca do fabricante)
*   [x] Estados Vazios (Empty States - "O sistema observa")
*   [x] Telas de Loading / Transições
*   [x] Erros Críticos (Kernel Panic)
*   [x] Documentos Oficiais / Exports

### Onde o Logo NÃO deve estar (O Território do Restaurante)
*   [ ] Cardápio Público (Menu Digital)
*   [ ] QR Codes na Mesa
*   [ ] Comandas de Impressão (Cozinha/Bar)
*   [ ] Interfaces de Cliente Final

### Forma
*   **Assinatura**: Ícone + Wordmark "ChefIApp™ OS"
*   **Destaque**: "OS" sempre em **Vermelho OS**.
*   **Ícone**: Monocromático (Ouro ou Brasa) dependendo do fundo.

---

## 2. Paleta Oficial — "Fire System"

A atmosfera não é "Dark Mode" genérico. É "Cozinha em Serviço".

### Cores-Base
| Nome | Hex | Função |
| :--- | :--- | :--- |
| **Preto Operacional** | `#050505` | Fundo absoluto, o vácuo onde a operação acontece. |
| **Carvão** | `#121212` | Superfícies, cards, painéis de controle. |
| **Brasa Escura** | `#2D0A0A` | Gradientes de fundo, profundidade (Vinho). |
| **Ouro Técnico** | `#C9A227` | Foco, seleção, linhas guia, logo. |
| **Vermelho OS** | `#D9381E` | **O NÚCLEO.** Ação crítica, identidade, irreversível. |

### A Regra do Vermelho
🔴 **O vermelho NUNCA é decorativo.**
Se virar botão comum, perde o poder de alerta.
Uso exclusivo para:
1.  Identidade do OS.
2.  Ações Irreversíveis (Deletar, Encerrar Dia).
3.  Erros Críticos.
4.  Estados de Alerta Real (Fila atrasada).

## 3. Tipografia & Presença Textual

### Headline (Títulos)
*   **Tom**: Forte, Declarativo, Sem Marketing.
*   **Fonte**: Outfit (Geométrica, técnica).
*   ❌ "Simplifique sua gestão"
*   ✅ **"Sistema operacional para restaurantes"**

### Corpo (Texto)
*   **Tom**: Curto, linguagem falada de cozinha (bater, rodar, liberar).
*   **Fonte**: Inter (Legível, funcional).
*   ❌ "Otimize seus processos para ganhar tempo"
*   ✅ **"Nada cai no esquecimento."**

## 5. Tom de Voz (A Personalidade)

Como o ChefIApp™ fala?
*   **Direto**: Não enrola.
*   **Calmo**: O caos já está na cozinha, o sistema é a âncora.
*   **Seguro**: Sabe o que está fazendo.
*   **Solene**: Respeita o ofício.

**Nunca seja**: Vendedor, "Coach", Infantil ("Parabéns!"), ou Corporativo.

> ❌ "Parabéns! Você está pronto para começar!"
> ✅ "Sistema criado. Operação liberada."

---

## 6. O Ritual (Onboarding)

Não é um "Wizard". É o nascimento do núcleo da operação.
*   **Visual**: Fogo Controlado (Gradiente "Ritual").
*   **Copy**: "Você está criando o núcleo. Essas decisões governam tudo."
*   **Estado**: Solene. Nada de barras de progresso gamificadas irrelevantes. Foco na consistência.

---

## 7. Comando Central (Dashboard)

*   **Visual**: "Ember" (Brasa). Menos gradiente vivo, mais carvão. Fundo escuro para não cansar a vista em ambientes com pouca luz.
*   **Estados Vazios**: Nunca diga "Você não tem vendas". Diga: **"O sistema só mostra o que existe. Inicie a operação."**
