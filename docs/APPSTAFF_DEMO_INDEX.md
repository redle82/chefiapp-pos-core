# 📚 AppStaff — Índice Completo de Demonstração

**Versão**: 1.0

## 🎯 Visão Geral

Este pacote completo permite demonstrar o AppStaff como um **sistema nervoso operacional**,
validando na prática que o restaurante funciona automaticamente, sem gestão manual.

**Mensagem central**: "O restaurante se move sozinho. As pessoas só acompanham."

---

## 📁 Estrutura de Arquivos

### Testes E2E

- **`tests/appstaff-full-operation-simulation.spec.ts`**
  - Teste Playwright completo
  - 15 funcionários simultâneos
  - 6 fases operacionais
  - Validações automáticas

- **`tests/README_APPSTAFF_SIMULATION.md`**
  - Documentação técnica do teste
  - Instruções de execução
  - Pré-requisitos

### Modo Demo Automático

- **`tests/appstaff-demo-mode.ts`**
  - Demo automático para apresentações
  - Executa fluxo completo automaticamente
  - Coleta métricas em tempo real
  - Uso: `npm run demo:appstaff`

### Métricas de Sucesso

- **`tests/appstaff-metrics-extractor.ts`**
  - Extrai métricas do teste E2E
  - Gera relatórios (Markdown + JSON)
  - Uso: `npm run metrics:appstaff`

### Documentação Comercial

- **`docs/APPSTAFF_DEMO_NARRATIVE.md`**
  - Narrativa comercial (sem código)
  - Explicação das 6 fases
  - Linguagem para vendas

- **`docs/APPSTAFF_DEMO_CHECKLIST.md`**
  - Checklist para QA humano
  - Validações por fase
  - Métricas de sucesso

- **`docs/APPSTAFF_SALES_SCRIPT.md`**
  - Script de venda ao vivo
  - Estrutura de apresentação (15-20 min)
  - Objeções comuns e respostas

## 🚀 Como Usar

### 1. Executar Teste E2E Completo

```bash
npx playwright test tests/appstaff-full-operation-simulation.spec.ts
```

**Resultado**: Validação completa do sistema em operação real.

---

### 2. Executar Demo Automático

```bash
npm run demo:appstaff
```

**Resultado**: Demonstração automática para apresentações comerciais.

---

### 3. Extrair Métricas

```bash
npm run metrics:appstaff
```

**Resultado**: Relatórios em `tests/reports/`:
- `appstaff-metrics-report.md`
- `appstaff-metrics.json`

---

### 4. Usar Script de Venda

1. Abrir `docs/APPSTAFF_SALES_SCRIPT.md`
2. Seguir estrutura de apresentação
3. Executar demo ao vivo
4. Usar narrativa comercial como apoio
