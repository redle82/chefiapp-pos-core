# PILOT BRIEF — ChefIApp v0.9

**Data**: 2026-01-04
**Duração**: 7 dias
**Tipo**: Piloto controlado (1 estabelecimento)

## O QUE É ESTE PILOTO

Um teste real de operação com pessoas reais, sem interferência técnica.

**Objetivo único**: Descobrir se o sistema funciona sem precisar explicar.

---

## O QUE NÃO É

| ❌ Não é | Porque |
|---|---|
| Demo de vendas | Não estamos vendendo, estamos validando |
| Teste de carga | 1 local, volume baixo |
| Beta público | Acesso fechado, controlado |
| Feature showcase | Só funcionalidades core |

---

## ESCOPO DO PILOTO

### ✅ Testar
- Criar pedidos
- Adicionar itens
- Pagar (cash/card)
- Abrir/fechar caixa
- Check-in de staff
- Completar tarefas

### ❌ Não testar
- Delivery
- Reservas
- Integrações externas
- Multi-local
- Gamificação

## O QUE OBSERVAR (NÃO PERGUNTAR)

### Momentos críticos
| Momento | Sinal de problema |
|---|---|
| Login | Hesita, digita errado |
| Criar pedido | Procura botão, demora |
| Adicionar item | Scroll confuso, categoria errada |
| Pagar | Não sabe qual botão |
| Task (staff) | Não entende o que fazer |

### Frases que revelam fricção
- "Onde fica...?"
- "Isso salvou?"
- "O que eu faço agora?"
- "Travou?"

👉 **Anote exatamente a frase.** Essa é a verdade.

---

## O QUE NÃO FAZER

| ❌ Evitar | Porque |
|---|---|
| Explicar antes | Esconde problemas de UX |
| Ajudar durante | Invalida o teste |
| Justificar bugs | Cria desconfiança |
| Prometer features | Gera expectativa falsa |

---

## COMO COLETAR FEEDBACK

### Durante (passivo)
- Observar sem interferir
- Anotar frases exatas
- Marcar momento + contexto

### Depois (ativo, dia 7)
Perguntas certas:
1. "O que foi mais fácil?"
2. "O que te travou?"
3. "O que faltou?"
4. "Usaria amanhã de novo?"

### Não perguntar
- "Gostou?" (viés de cortesia)
- "Ficou bonito?" (irrelevante)
- "Faz sentido?" (sim envergonhado)
