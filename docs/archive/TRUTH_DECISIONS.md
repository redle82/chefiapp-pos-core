# TRUTH DECISIONS (ADR) — v1.0.0

**Date**: 2025-12-25
**Status**: RATIFIED
**Context**: Post-Audit "Antigravity Deep Dive"

This document records the architectural decisions regarding the "Truth Consistency" of the ChefIApp ecosystem for the v1.0.0 release. It serves as a transparent admission of system boundaries.

---

## 001. AppStaff as "Prototype Preview"
**Context**: The `AppStaff` module currently operates on 100% mock data and is disconnected from `useOfflineQueue` and `useCoreHealth`.
**Decision**: In v1.0.0, AppStaff is classified as a **Visual Prototype / Technology Preview**.
**Consequence**:
- It must be explicitly labeled (e.g., "Preview", "Demo", or "Coming Soon") in the UI to prevent users from relying on it for critical operations.
- Data loss on refresh is expected behavior for this version.
- **Truth Status**: ⭕ **FICTION** (Accepted for UI showcase, rejected for Ops).

## 002. TPV as "Single Source of Operations"
**Context**: The TPV module is the only component fully integrated with the Offline Truth Layer.
**Decision**: TPV is the **only** supported operational interface for v1.0.0.
**Consequence**:
- Merchant Portal Dashboard is treated as a "Setup Launcher", not an operational dashboard.
- Operational metrics in the Dashboard are static mocks and do not reflect TPV activity.
- **Truth Status**: ✅ **TRUTH** (Gold Standard).

## 003. Public Page & Dashboard as "Static Mocks"
**Context**: These pages use hardcoded data and do not consume the configuration generated by the Setup Wizard.
**Decision**: Accepted as "Static Templates" for v1.0.0 to demonstrate capability.
**Consequence**:
- Users completing setup will see their data persisted in the API, but *not* reflected in the Dashboard/Public views immediately.
- This "Split Brain" is acknowledged as a known limitation to be resolved in v1.1.0.
- **Truth Status**: ⚠️ **PARTIAL TRUTH** (Data exists in backend, UI shows template).

## 004. Landing Page "AI Building"
**Context**: The onboarding flow uses a 2.5s artificial delay labeled "AI Building infrastructure...".
**Decision**: Retained as "Marketing Aesthetics" (User Experience).
**Consequence**:
- We acknowledge this is a "White Lie" for pacing and perceived value.
- It does not violate "Truth Zero" because it does not block critical paths or hide failures (Health Check is real).
- **Truth Status**: 🎭 **THEATER** (Accepted).
