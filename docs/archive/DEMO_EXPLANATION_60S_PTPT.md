# ChefIApp — Explicação para Demo (60s) — PT-PT

> Objetivo: explicação clara, falável e não-técnica para mostrar páginas + sistema (não “pitch de investidor”).

## Roteiro final (~60s)

“Olá — obrigado por assistir.
Vou mostrar rapidamente como isto funciona na prática.”

(Abro a página pública: /public/sofia-gastrobar)

“Isto é uma página própria do restaurante.
Não é um marketplace, não é um template genérico.
Aqui o restaurante tem site, menu e pedidos, tudo no mesmo sítio.”

“Repare no nome, no hero, na URL própria com o slug e no menu já visível.
Isto já é o produto real, não é uma demo falsa.”

(Faço scroll no menu e paro num item)

“E aqui não estamos a falar só de presença online.
Cada item é operacional — tem preço, lógica de pedido e fluxo real.
O objetivo não é mostrar. É receber pedidos reais.”

(Abro o Merchant Portal e clico “Criar pedido teste (1 item)”)

“Agora vou criar um pedido verdadeiro.
Isto não é simulação: é idempotente, tem regras e entra no fluxo operacional.”

“Se houver pagamento ligado, o pedido segue.
Se não houver, o sistema bloqueia — de forma clara.
Pedido criado: o sistema responde como em produção.”

(Volto ao portal e aponto o wizard)

“Tudo isto é configurado aqui, numa única página, em cinco passos:
Identidade, Menu, Pagamentos, Design e Publicar.”

“Cada passo tem estado: concluído, atual, pendente —
e quando algo está bloqueado, aparece como bloqueado.”

(Entro num passo bloqueado)

“Quando algo está fora do plano, o sistema não quebra.
Ele explica.
O bloqueio vira orientação — não frustração.”

(Clico “Ver planos”)

“Aqui está a explicação completa.
O sistema vende como funciona: configura → publica → pede → opera.”

(Fecho)

“Não é um site.
Não é um plugin.
É o ciclo completo do restaurante, sem intermediários.”

## Porque esta versão funciona
- Cabe confortavelmente em ~60s
- Fala de realidade, não de promessa
- Mostra páginas, não slides
- Explica bloqueios como inteligência, não limitação
- Serve para cliente e para parceiro
