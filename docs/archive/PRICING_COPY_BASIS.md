# ChefIApp — Pricing Copy (BASIC / PRO / EXPERIENCE)

> Objetivo: copy comercial alinhada com o que o produto já faz hoje e com gates reais (sem inventar feature).

## BASIC — “Entrar no jogo”
**Promessa:** Página publicada + pedidos diretos (sem marketplace).

**Inclui:**
- Página web publicada (`/public/:slug`) com menu e CTA
- Pedidos web criam ordem real no core
- Pagamento direto do restaurante (quando configurado)

**Quando faz upgrade:**
- Quer design/experiência mais “marca”
- Quer elevar a conversão (e parecer premium)

## PRO — “Operação + design profissional”
**Promessa:** A melhor experiência para vender direto com cara de marca.

**Inclui:**
- Tudo do BASIC
- Nível PRO de web/design (controle via wizard)
- Experiência de admin mais clara: passos, status e bloqueios visíveis

**Quando faz upgrade:**
- Quer diferenciar marca e aumentar ticket
- Quer recursos premium sem depender de marketplace

## EXPERIENCE — “Marca própria e crescimento”
**Promessa:** Experiência premium (white-label) e base para crescimento.

**Inclui (hoje, no core do produto):**
- Tudo do PRO
- Nível EXPERIENCE (gated)

**Narrativa (roadmap natural, sem prometer datas):**
- White-label mais profundo, domínio/experiência premium
- Fidelidade e campanhas conectadas ao core operacional

## Mensagens de bloqueio (in-product)
Quando o plano bloquear (`FEATURE_BLOCKED` / `WEB_LEVEL_BLOCKED`), a UI deve comunicar:
- "🔒 Recurso do plano PRO/EXPERIENCE — faça upgrade para publicar neste nível"
- CTA: "Ver planos" / "Falar conosco"

## 1-liner por plano
- BASIC: “Publica e recebe pedidos diretos.”
- PRO: “Vende direto com cara de marca.”
- EXPERIENCE: “Experiência premium para crescer.”
