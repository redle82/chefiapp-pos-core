## 📝 Script Completo

### 1. Problema (1 min)

**"Você tem 3 problemas principais no restaurante:**

**1. Atrasos na produção**
- Não sabe quando item está atrasado
- Descobre só quando cliente reclama
- Sem sistema de alerta automático

**2. Estoque desorganizado**
- Não sabe quando comprar
- Descobre só quando falta
- Sem conexão com produção

**3. Menu é catálogo**
- Menu não define comportamento do sistema
- Tempo de preparo não existe
- Sistema não sabe como operar

**Resultado: atrasos, desperdício, ineficiência."**

---

### 2. Solução (2 min)

**"ChefIApp resolve isso conectando tudo:**

**1. Menu como contrato operacional**
- Cada item define tempo de preparo (obrigatório)
- Cada item define estação (obrigatório)
- Sistema usa isso para tudo:
  - KDS mostra tempo real
  - Task Engine detecta atrasos
  - Cliente vê status preciso

**2. Task Engine automático**
- Detecta atrasos automaticamente (>120% do tempo)
- Gera tarefas de estoque (quando abaixo do mínimo)
- Gera tarefas de entrega (quando pedido pronto há 5+ min)
- **Fecha tarefas sozinhas** (quando condição some)

**3. Estoque conectado à operação**
- Consome automaticamente (via BOM)
- Gera lista de compras automática (quando abaixo do mínimo)
- Confirma compra → atualiza estoque → fecha tarefas
- **Loop fechado** (compra → estoque → produção → compra)

**Resultado: sistema detecta problemas antes que virem crise."**

---

### 3. Diferencial (1 min)

**"Não competimos com Toast ou Square por features. Competimos por arquitetura.**

**Toast / Square / Lightspeed:**
- Focam em gestão (POS, delivery, marketing)
- Task Engine não existe ou é genérico
- Estoque desconectado da operação
- Menu é catálogo

**ChefIApp:**
- Foca em produção (não gestão genérica)
- **Task Engine automático (único)**
- **Estoque conectado à operação (único)**
- **Menu como contrato operacional (único)**

**👉 Não vendemos features. Vendemos sistema operacional."**
