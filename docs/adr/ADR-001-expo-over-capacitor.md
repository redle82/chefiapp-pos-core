# ADR-001: Expo over Capacitor for Mobile App

**Status:** Accepted
**Date:** 2026-02-09
**Decision Makers:** GoldMonkey (Founder / Lead Dev)

## Context

ChefIApp POS requires a cross-platform mobile app (iOS + Android) for
restaurant staff: waiters, kitchen, bar, and managers. The app needs:

- TCP socket printing (ESC/POS thermal printers)
- Offline-first with background sync
- Push notifications
- Biometrics / Secure Store
- Real-time WebSocket (KDS sync)
- Haptic feedback

Two frameworks were evaluated: **Expo (React Native)** and **Capacitor (Web ↔ Native bridge)**.

## Decision

**Expo** was chosen as the definitive mobile framework.

## Rationale

| Criterion                 | Expo                                                  | Capacitor                                             |
| ------------------------- | ----------------------------------------------------- | ----------------------------------------------------- |
| **OTA Updates**           | `expo-updates` — ship JS fixes without App Store      | Requires full rebuild                                 |
| **Managed Workflow**      | EAS Build handles signing, provisioning               | Manual Xcode / Gradle setup                           |
| **Native Modules**        | Expo Modules API + prebuild                           | Capacitor plugins (smaller ecosystem for niche needs) |
| **TCP Sockets**           | `react-native-tcp-socket` works via custom dev client | Requires Cordova plugin bridge                        |
| **Typed Router**          | `expo-router` with typed routes                       | `@ionic/react-router` (less type-safe)                |
| **Developer Experience**  | `expo start` → instant QR code                        | `npx cap run` → full native build each time           |
| **Community / Ecosystem** | Largest React Native ecosystem                        | Growing but Web-first mindset                         |
| **New Architecture**      | Full support (Fabric, TurboModules)                   | Not applicable                                        |

### Key trade-off accepted

Expo requires a **custom development build** (`expo prebuild`) for native modules
like `react-native-tcp-socket`. This is already configured in `eas.json` and
`app.json` (`newArchEnabled: true`).

## Evidence in Codebase

- `mobile-app/app.json` — Expo config with `expo-router`, `@sentry/react-native`
- `mobile-app/package.json` — `expo ~54.0.32`, no `@capacitor/*` dependencies
- `mobile-app/eas.json` — EAS Build profiles (development, preview, production)
- `mobile-app/app/(tabs)/` — File-based routing via `expo-router`
- Android/iOS native dirs generated by `expo prebuild`, not maintained manually

## Consequences

- All mobile builds go through **EAS Build** (CI or local `eas build`)
- Native modules must be compatible with Expo's custom dev client
- `expo prebuild` regenerates `android/` and `ios/` — never edit Xcode/Gradle directly
- Capacitor remnants in `merchant-portal/` (used for PWA bridge) stay as-is; the
  **mobile-native app** is Expo-only
