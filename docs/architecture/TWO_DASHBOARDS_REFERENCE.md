# Dois dashboards — referência única

**~60 s.** Contrato: [COGNITIVE_MODES_OWNER_DASHBOARD.md](./COGNITIVE_MODES_OWNER_DASHBOARD.md).

**Decisão estrutural:** Existe 1 dashboard real (Web). O **dashboard central** é o Owner Dashboard em **`/admin/reports/overview`** (shell Last.app-like: sidebar + header + este conteúdo). Não criar dashboards paralelos. Não mover para outra rota. Sidebar = mapa, não dashboard.

---

## 1. Quantos?

Um dashboard real (Web). Uma visão comprimida (App). Não dois dashboards concorrentes. Não meio termo.

---

## 2. Quais?

- **Staff** — App. Tempo real. Agir agora.
- **Owner** — Web (completo) + App (compacto). Entender e decidir.

---

## 3. Onde?

| Superfície | O que existe |
|------------|--------------|
| **App** | Staff (principal). Owner (compacto), só Owner; acesso “Visão do Dono”. |
| **Web** | Um dashboard real. **Central:** `/admin/reports/overview` = Owner Dashboard (Dinheiro / Operação / Pessoas / Risco / Feed). Relatórios e configuração fora do centro. |

**Sidebar do portal:** Só navega (mapa do sistema). Nunca mostra métricas. Configuração no rodapé da sidebar.

---

## 4. Para quem?

- **Staff** — Quem opera. Tempo: **Agora**.
- **Owner** — Dono. Tempo: **Entender / decidir**.

---

## 5. O que não são?

Dois dashboards concorrentes. Owner ≠ Staff com mais números. Sidebar ≠ dashboard (sidebar = mapa). Não criar dashboards paralelos. Não mover o Owner Dashboard para outra rota — ele é o miolo central em `/admin/reports/overview`. Misturar os dois tempos numa tela confunde e destrói foco.

---

## 6. Regra

Não misturar tempos mentais. Staff = agir. Owner = ler e decidir.
