# Core Manifesto — ChefIApp

Princípios de governança do sistema. Memória institucional; não substitui contratos.

---

## Governança operacional (ORE)

**ORE governa por semântica; UI reflete decisões; layout nunca decide.**

- O ORE é a autoridade única que decide se uma superfície pode operar e que directiva de UI aplicar.
- A UI (sidebar, páginas, BlockingScreen) executa as decisões do ORE; não decide prontidão nem organiza o sistema.
- Layout, copy e fluxos podem evoluir sem alterar o ORE, desde que respeitem as suas decisões.

Referência: [OPERATIONAL_READINESS_ENGINE.md](../bootstrap/OPERATIONAL_READINESS_ENGINE.md) — secção 10 (ORE e o Sidebar).
