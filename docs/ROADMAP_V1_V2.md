## 🔮 v2 — Evolução Baseada em Uso Real (Q2 2026)

### Prioridade Alta (Baseado em Necessidade Real)

#### 1. Relatórios Básicos
**Por que:**
- Necessário para gestão
- Dados já existem
- Complexidade média

**O que inclui:**
- Vendas por período
- Produtos mais vendidos
- Ticket médio
- Exportar para Excel/CSV

**Estimativa:** 2-3 semanas

---

#### 2. Histórico de Pedidos
**Por que:**
- Necessário para consulta
- Dados já existem
- Complexidade baixa

**O que inclui:**
- Lista de pedidos fechados
- Filtros (data, mesa, origem)
- Detalhes do pedido
- Reabrir pedido (se necessário)

**Estimativa:** 1-2 semanas

---

#### 3. Melhorias no Task Engine

**Estimativa:** 2-3 semanas

## 🎯 Critérios para v2

### O Que Define Prioridade

1. **Necessidade Real**
   - Problema identificado em uso real
   - Bloqueador para operação
   - Feedback consistente

2. **Complexidade vs. ROI**
   - Complexidade baixa + ROI alto = Prioridade Alta
   - Complexidade média + ROI alto = Prioridade Média
   - Complexidade alta + ROI baixo = Prioridade Baixa

3. **Dados Disponíveis**
   - Se dados já existem = Mais fácil
   - Se precisa coletar = Mais difícil

---

## 📊 Processo de Decisão (v1 → v2)

### 1. Coletar Feedback (Uso Real)
- Problemas encontrados
- Features solicitadas
- Melhorias sugeridas

### 2. Priorizar (Baseado em Critérios)
- Necessidade real
- Complexidade vs. ROI
- Dados disponíveis

### 3. Implementar (Iterativo)
- Feature por feature
- Validar com usuários
- Iterar baseado em feedback
